# MES System for Fiberglass Pipes Production (Telegram Bot + Mini App)

## 📌 Описание проекта
Легковесная система управления производством (MES) стеклопластиковых труб. 
Интерфейс: Telegram Bot (управление стадиями) + Telegram Web App (ввод данных сырья и ОТК).

## 🛠 Технический стек
- Python 3.11+, aiogram 3.x (строго асинхронный подход)
- FastAPI, HTML5, TailwindCSS, Telegram Web App API
- PostgreSQL + SQLAlchemy 2.0 (asyncio) + Alembic

## 👥 Ролевая модель (7 ролей по Telegram ID)
1. **Администратор:** Бот. Постановка задач на партию.
2. **Инженер ОТК:** Mini App. Генерация серийных номеров, поэтапный контроль, разрешение на стадии, финальная приемка.
3. **Начальник смены:** Бот. Управляет очередью труб, дает СТАРТ/СТОП всем 6 производственным стадиям.
4. **Дозировщик:** Mini App. Отвечает за "мокрую" химию (Смола, Октоат кобальта 12%, Акперокс). Вводит фактический расход по стадиям.
5. **Лаборант:** Mini App. Отвечает за тесты (время гелеобразования, впитываемость).
6. **Технолог:** Mini App. Отвечает за "сухие" материалы (Стеклоровинг, Вуаль, UD-ленты, Песок).
7. **Операторы:** Физически выполняют работу (пользуются ботом только для старта/стопа по поручению Начальника смены).

## 🏭 СТРОГИЙ БИЗНЕС-ПРОЦЕСС (Workflow)
ИИ обязан строго следовать этой последовательности для каждой трубы:

**ШАГ 0. Задача и Идентификация**
- Админ создает задачу: DN, PN, SN, Длина, Тип (с песком/без), Кол-во слоев песка (1 или 2), Наличие раструба/ниппеля (да/нет - бывают прямые трубы).
- ОТК генерирует уникальный Серийный номер для каждой трубы в задаче.

**ШАГ 1. Лайнер (Liner)**
- Дозировщик: проверяет рецептуру.
- Начальник смены: СТАРТ стадии Лайнер.
- Лаборант: во время формовки берет образец смолы+акперокса, фиксирует Фактическое время гелеобразования.
- Начальник смены: СТОП стадии Лайнер.

**ШАГ 2. Сушка после Лайнера + Списание сырья Лайнера**
- Труба уходит на Сушку (СТАРТ сушки).
- Дозировщик вводит ФАКТ химии на Лайнер (Изофталевая смола, Кобальт, Акперокс).
- Технолог вводит ФАКТ сухих материалов на Лайнер (Полиэфирная марля, Вуаль, Сшитый материал, UD300).
- СТОП сушки.

**ШАГ 3. Виндер (Winder) + Песок**
- Начальник смены: СТАРТ стадии Виндер.
- Лаборант (во время намотки): 
  1) Тест времени гелеобразования (из намотки стеклоровинга).
  2) Тест впитываемости и однородности (из смеси смола+кобальт+акперокс+песок).
- ОТК (во время намотки): замеряет и фиксирует фактическую толщину песчаных слоев (если труба с песком).
- Дозировщик: вводит ФАКТ химии для Виндера ОТДЕЛЬНО для стекловолокна и ОТДЕЛЬНО для песка (Песок 1, Песок 2).
- Технолог: вводит ФАКТ сухих материалов Виндера (Стекловолокно 2400tex, Песок, UD250, марля для песка).
- Начальник смены: СТОП стадии Виндер.

**ШАГ 4. Сушка после Виндера и Разрешение ОТК**
- Начальник смены: СТАРТ/СТОП сушки после Виндера.
- Инженер ОТК: Оценивает степень сушки и в системе нажимает кнопку "Разрешение на токарную обработку". Без этого шага Токарь не начнется.

**ШАГ 5. Токарный станок (Turning)**
- *Условие: Если труба "Прямая" (без раструба/ниппеля), эта стадия пропускается.*
- Начальник смены: СТАРТ/СТОП обработки.
- Инженер ОТК вводит геометрию: Длина окружности трубы, Длина окружности раструба, Толщина стенки, Толщина стенки раструба, Наружный диаметр ниппеля, Диаметр каналов (1 или 2), Глубина каналов, Ширина каналов, Длина обработанной части.

**ШАГ 6. Трубосъем (Extraction) и Финальный ОТК**
- Если геометрия из Шага 5 в норме, Начальник смены: СТАРТ/СТОП трубосъема.
- Инженер ОТК: Проводит визуальный контроль (внутри и снаружи на дефекты). Выставляет финальный статус: "Годен к выпуску" / "Брак".

## 🤖 ПРАВИЛА ДЛЯ AI ПРИ НАПИСАНИИ КОДА (БД)
1. База данных (SQLAlchemy 2.0) должна идеально отражать этот Workflow.
2. `Tasks` - таблица общих заказов. `Pipes` - конкретные трубы с серийниками (связь One-to-Many).
3. `ProductionStages` - таймстемпы (start_time, end_time) для всех 6 стадий (Liner, LinerDrying, Winder, WinderDrying, Turning, Extraction).
4. Раздели учет материалов на: `ChemistryLogs` (Дозировщик) и `DryMaterialLogs` (Технолог).
5. `LabTests` - таблица для Лаборанта (гелеобразование лайнера, виндера, тест песка).
6. `QCPassports` - таблица для ОТК (толщины, разрешения на токарку, геометрия токарки, финальный статус).
7. Используй `Mapped` и `mapped_column`, типы данных `DateTime(timezone=True)`, `Float` для размеров и веса.
8. Пиши код только на английском, комментарии на русском. Без заглушек, пиши полные модели.
